# SASS IETF Internet-Draft 遞交清單

> 逐項確認後打勾 `[x]`，全部完成即可遞交。

---

## 一、前置準備

- [ ] 已在 <https://datatracker.ietf.org/> 註冊帳號
- [ ] 已驗證註冊電子郵件
- [ ] 已填寫 [`AUTHOR_INFO.md`](./AUTHOR_INFO.md) 中的作者資訊

## 二、文件確認

- [ ] `draft-sakistudio-sass-00.txt` 已存在於本目錄
- [ ] 確認文件標題為 `SASS: SSH Agent Signature Scheme`
- [ ] 確認文件中的作者姓名與 `AUTHOR_INFO.md` 一致
- [ ] 確認文件中的 email 與 `AUTHOR_INFO.md` 一致

## 三、格式檢查（選做但建議）

- [ ] 使用 [IETF I-D 格式檢查工具](https://author-tools.ietf.org/) 預檢文件
- [ ] 無嚴重格式錯誤（warnings 可接受）
- [ ] 每行不超過 72 字元
- [ ] 頁碼與目錄正確

## 四、遞交操作

- [ ] 前往 <https://datatracker.ietf.org/submit/>
- [ ] 選擇 **"Upload a new Internet-Draft"**
- [ ] 上傳 `draft-sakistudio-sass-00.txt`
- [ ] 選擇 **"Individual submission"**
- [ ] 確認作者資訊正確
- [ ] 點擊提交

## 五、遞交後確認

- [ ] 收到格式檢查結果 email
- [ ] 在 <https://datatracker.ietf.org/doc/draft-sakistudio-sass/> 確認 Draft 已上架
- [ ] 記錄遞交日期：`________`（有效期 6 個月）

## 六、RFC 推進準備

> 詳細策略請參閱 [`RFC_STRATEGY.md`](./RFC_STRATEGY.md)

### 6.1 加入社群

- [ ] 訂閱 `curdle` 郵件列表（<https://www.ietf.org/mailman/listinfo/curdle>）
- [ ] 潛水觀察 1-2 週，了解社群討論風格
- [ ] 在郵件列表發布 Draft 介紹（範本見 `RFC_STRATEGY.md` §三）

### 6.2 聯繫 Area Director

- [ ] 查詢當前 SEC Area Director（<https://www.ietf.org/about/groups/iesg/>）
- [ ] 寄送 email（範本見 `RFC_STRATEGY.md` §二）
- [ ] 收到回覆，確認推進路線

### 6.3 IETF Meeting

- [ ] 確認下一場 IETF Meeting 日期（<https://www.ietf.org/how/meetings/>）
- [ ] 向 WG Chair 申請議程時段（會議前 6 週）
- [ ] 準備 10-15 分鐘簡報
- [ ] 參加會議（遠端或現場）

### 6.4 WG Adoption

- [ ] 收到足夠社群正面反饋
- [ ] WG Chair 發起 Adoption Call
- [ ] Adoption Call 通過
- [ ] Draft 更名為 `draft-ietf-curdle-sass`

### 6.5 最終推進

- [ ] 完成 WG 內部 review 與修訂
- [ ] 通過 WG Last Call
- [ ] 通過 IESG Review + IETF Last Call
- [ ] RFC Editor 編輯完成
- [ ] 🎉 發布為 RFC

---

> **提示**：第一次遞交只需完成 一～五。第六項為 RFC 推進路線，可在 I-D 遞交成功後逐步執行。
