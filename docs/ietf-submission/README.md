# SASS IETF Internet-Draft 遞交指南

> **目標**：讓你只需要 **填寫姓名/email → 上傳檔案 → 點提交**，其餘全部已準備好。

---

## 前置作業（一次性）

1. 前往 <https://datatracker.ietf.org/> 註冊帳號
2. 確認電子郵件已驗證
3. 填寫 [`AUTHOR_INFO.md`](./AUTHOR_INFO.md) 中的作者資訊

## 遞交步驟

1. 前往 <https://datatracker.ietf.org/submit/>
2. 選擇 **"Upload a new Internet-Draft"**
3. 上傳 `draft-sakistudio-sass-00.txt`（位於本目錄）
4. 選擇 **"Individual submission"**
5. 確認作者資訊（應與 `AUTHOR_INFO.md` 一致）
6. 提交

## 遞交後

- 自動格式檢查結果會寄到你的 email
- Draft 會出現在 <https://datatracker.ietf.org/doc/draft-sakistudio-sass/>
- 有效期 **6 個月**，到期前可以提交更新版（`-01`, `-02`, ...）

## 如果要推動成為正式 RFC

有三條路徑可選：

| 路徑 | 說明 | 難度 |
|------|------|------|
| **Working Group** | 聯繫 `secsh` 或 `secdispatch` Working Group，請求採納 | ★★★ |
| **Independent Stream** | 走 Independent Submission Stream (ISE)，由 ISE 編輯審查 | ★★ |
| **保持為 I-D** | 僅維持 Internet-Draft 狀態，持續更新即可（很多協定都是這樣） | ★ |

> **建議**：先以 Individual I-D 遞交，觀察社群反饋後再決定路徑。
