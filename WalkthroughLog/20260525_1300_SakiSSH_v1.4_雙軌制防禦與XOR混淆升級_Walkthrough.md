## 🛠️ 實作變更詳情

### 1. 雙軌制防禦落地 (Dual-Standard Active Defense)
我們在 `saki-ssh-daemon/src/tarpit.rs` 中新增了 `vi_swap` 反制機制，並修改了 `src/v6_integration.rs` and `src/main.rs` 中的 13Policy 攔截路徑，將防禦邏輯區分為雙軌：
* **Verified Internal Agent (已認證內部 Agent)**：
  * **觸發點**：身分已通過認證 (`identity_pubkey != "anonymous"`) 但執行了 13Policy 違規指令 (例如 `rm -rf /`)。
  * **反制手段**：**Vi-Swap 防禦**。回傳一組專門設計的靜態 `terminal escape sequence`，在 stdout 渲染一個逼真的 Vi/Vim 編輯器初始介面並隱藏游標，使 LLM 的 stdout 解析程序在解析到這些 terminal 序列時卡死或優雅掛起，困在 Vi 狀態中，而不耗費任何多餘的 40MB 頻寬或大額 API Token 預算。
* **Unverified External Agent (未授權外部 Agent)**：
  * **觸發點**：身分未通過認證或匿名 (`identity_pubkey == "anonymous"`) 且執行了危險指令。
  * **反制手段**：**Zero-Allocation Tarpit (焦油坑)**。回傳由共享唯讀高熵 Buffer 構成的 40MB 慢速垃圾資料流，在 O(1) 空間開銷下強制反向耗竭外部攻擊者的 Token Context 與資源。

### 2. LocalHost XOR 混淆與微隨機化防禦升級
我們升級了 `saki-ssh-daemon/src/localhost_defense.rs`，將原本靜態的欺騙資訊轉變為高仿真、抗檢測的動態防護：
* **儲存空間與記憶體 (df / free) 微幅動態隨機化**：
  * 每次探測時，隨機在微小區間調整 Used/Free 記憶體與可用磁碟 Block 數量，模擬活體 OS 的系統資源波動，徹底杜絕惡意 Agent 透過「靜態內容特徵比對」識破假檔案系統。
* **憑證與環境變數 (credentials / env) 動態 XOR 混淆**：
  * 當探測 `.aws/credentials` 時，使用動態隨機產生的金鑰進行高強度 XOR 混淆處理，生成看似合法的偽造憑證，使試圖盜取金鑰的 Agent 在嘗試解密與分析時消耗大量的算力與 Token，大幅提高攻擊成本。
