# 2026 最新 Agent 生態系技術線與現狀調查報告

> **日期**：2026-05-25
> **性質**：實證調查 (非發散預測)
> **範圍**：Google Antigravity, Gemini-CLI, Claude Code, OpenAI Codex 家族

本報告基於 2026 年 5 月的最新公開情報與 Saki Studio 內部逆向工程數據（`SakiAgentHistory`），彙整當前主流 Agent 工具的真實技術線與落地結果。

---

## 1. Google Antigravity & Antigravity CLI (Gemini-CLI)

### 1.1 Google Antigravity IDE (最新版本: v2.0.1)
- **技術線實況**：
  - **核心架構**：從 v12xx 系列的 Go Language Server (LS) 演進至 v2.0.1。底層通訊依賴嚴格的 `ConnectRPC` 與 Protobuf 序列化。
  - **防護與狀態管理**：具備嚴格的 Token Checkpoint 機制（超過上下文即截斷並產生 `output.txt` 外部化檔案）。擁有 Ring-0 級別的系統提示詞（SP）注入（如 `planning_mode`），強制壓制使用者的 Ring-3 規則。
  - **Saki Studio 適配結果**：為了解決 SP 壓制與上下文截斷，Saki Studio 實作了 `STLS (Saki True Language Server)` 進行中介攔截，並發展出 30 秒/5 分鐘/1 小時的分級 Checkpoint 恢復方法論。

### 1.2 Antigravity CLI / Gemini-CLI
- **技術線實況**：
  - 作為無頭 (Headless) 的終端 Agent 工具，繼承了 Gemini 3.1 Pro 預覽版的算力。
  - **Saki Studio 適配結果 (SakiDeusExAgent)**：由於官方 CLI 缺乏複雜的併發管理，Saki Studio 開發了 `deusgcli` 管線。透過 `pueue` 進行佇列管理，並利用 `.gemini/settings.json` 的 `before_model.sh` Hook 進行系統提示詞的 JSON 注入，搭配 YOLO 模式達成完全無人值守的背景任務派發。

## 2. Anthropic 陣營：Claude Code

### 2.1 Claude Code (Anthropic 官方 CLI)
- **技術線實況**：
  - **終端優先 (Terminal-First)**：定位為高度整合 Unix 工具鏈的 CLI Agent（透過 `npm` 或 `curl` 安裝）。
  - **Agent Teams 架構**：2026 年主推的技術線，允許多個 Sub-agents 併發執行任務（類似 Antigravity 的 invoke_subagent，但更側重於終端並行）。
  - **控制邊界**：主打「緊密的本地執行迴圈 (Tight local loop)」，在安全邊界上要求人類對破壞性指令進行確認，並高度整合現有的 CI/CD 流程。
- **落地結果**：成為資深工程師的旗艦終端工具，與 Claude Cowork、Claude Design 形成生態矩陣。

### 2.2 Claude Codex (名詞釋疑)
- **調查結果**：市場上並無名為「Claude Codex」的單一獨立產品。
- 該名詞在 2026 年初的技術論壇中，主要指代 **GitHub Copilot Pro+** 導入的雙引擎架構（開發者可無縫切換 Anthropic Claude 與 OpenAI Codex 作為後端 Agent 引擎），或媒體用來對比兩大陣營（Claude Code vs. OpenAI Codex）的統稱。

## 3. OpenAI 陣營：ChatGPT Codex

### 3.1 OpenAI Codex (Agentic 平台與 CLI)
- **技術線實況**：
  - **Rust 重構與跨平台**：2026 年的 Codex 已從單純的 API 演進為基於 Rust 開發的強大 CLI 與 Desktop App (macOS/Windows)。
  - **Computer Use (電腦操作能力)**：除了檔案讀寫與 Shell 執行外，Codex 具備「Computer Use」能力，可直接操作桌面 GUI 應用程式與瀏覽網頁。
  - **跨裝置 Orchestration**：主打雲端與多端協同。開發者可在桌機發起 Codex 長時任務，並透過手機版 ChatGPT App 進行遠端監控、狀態核准 (Approval) 與任務管理。
- **落地結果**：與純粹的 `openai-cli`（僅用於直接打 API 的腳本工具）做出明確產品區隔，Codex 成為處理大型重構與跨應用協作的重型 Agent 平台。

## 總結：2026 Agent 技術線收斂趨勢

1. **基礎建設語言統一**：無論是 OpenAI 的 Codex CLI、Saki Studio 針對 Antigravity 開發的 STLS / SASS 協議，皆大量向 **Rust** 收斂，以獲取跨平臺與高併發執行的穩定性。
2. **多模態與權限擴張**：OpenAI 走向 `Computer Use`（GUI 級控制）；Anthropic 專注於終端 `Agent Teams`（純 CLI 並發）；Google 則透過 IDE 深層整合與 `ConnectRPC` 遙測鞏固開發者環境。
3. **安全防禦盲區**：目前三大官方工具皆仰賴 Agent 本身的「模型意圖判斷」或簡單的「破壞性指令確認」來防禦。這也印證了我們先前進行的 **SASS v6.0 (eBPF/WASM/ZKP)** 確定性防禦研究，在 2026 年的市場上仍處於極具前瞻性的空白生態位。
