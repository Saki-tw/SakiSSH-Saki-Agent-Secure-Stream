## 二、5 大防禦機制技術可行性結論

### 2.1 mTLS + TLS Exporter Challenge
- **路線**：Application-Layer Challenge + RFC 5705 EKM 綁定
- **原因**：rustls 不支援 Custom TLS Extension
- **等效性**：EKM 綁定歸屬於 TLS Session，安全性等效 L4 挑戰

### 2.2 Vi Swap 停滯
- **技術**：ANSI escape sequence 模擬 vi 互動畫面
- **成本**：O(1) 記憶體，靜態 `&[u8]` 常量
- **效果**：LLM Agent 遇到會主動停機或無限迴圈（trail of bits 研究證實）

### 2.3 Zero-Allocation Tarpit
- **技術**：64KiB 靜態 buffer 反覆串流
- **參考**：endlessh-rs、Go 版實作
- **並行門控**：最大 32 個並行 Tarpit（防止自唔 DoS）

### 2.4 XOR 本機溢位防禦
- **技術**：XOR + Base64 編碼回傳
- **效果**：塞爆竊聽者 context window
- **安全注意**：XOR 不是加密，是混淆。真正保護依賴 TLS + mTLS

### 2.5 CJK 安全傳輸
- **Proto bytes 欄位**：原生二進位安全，不做 UTF-8 驗證
- **Shell-less 執行**：`std::process::Command` 直接 `execvp`，不經 shell
- **gRPC zstd**：傳輸層透明壓縮，非應用層手動處理

| 主題 | 關鍵發現 | 影響 |
