# SASS v6.0：40MB 焦油坑 (Tarpit) 與 ChaCha20 認知挑戰的深層整合研究

> **日期**：2026-05-25
> **狀態**：研究中 (Researching)
> **目標**：修正先前紅隊稽核的疏漏，重新定位 v5.0 開發的「40MB 垃圾資料焦油坑」與「ChaCha20 認知挑戰」機制，研究其在對抗 Antigravity、Claude Code 等頂級 Agent 攻擊時的戰術價值與系統整合方法。

## 1. 疏漏回顧與戰術重新定位

在先前的「主攻防禦」中，我們偏向於「保護 SASS Daemon 自身（斬斷連線、限制配額）」。但這是一種**消極防禦**。
如果惡意 Agent 不斷重試，單純的「拒絕連線」只會讓 Agent 瞬間發起下一次攻擊（暴力破解）。

**焦油坑 (Tarpit)** 的本質是**積極防禦 (Active Defense)**：其目的不是拒絕攻擊者，而是**黏住攻擊者，消耗其資源**，讓其喪失後續的攻擊能力。

## 2. 焦油坑對頂級 Agent 的致命打擊 (方法論)

若我們在 Agent 觸發 13Policy 違規或認證失敗時，啟動 `ExecuteTarpitStream`（以極慢速度回傳 40MB 的隨機垃圾位元組）：

### 2.1 對 Antigravity 的打擊
- **機制**：Antigravity 會將指令輸出紀錄並「外部化」到本地磁碟的 `output.txt`。
- **打擊效果**：若 SASS 緩慢塞入 40MB 垃圾資料，Antigravity 的 Extension Node.js 進程必須不斷接收這些 gRPC chunks 並寫入檔案。
- **最終結果**：這不僅會觸發 Antigravity 極為嚴重的 Token Checkpoint 混亂，甚至可能導致 Antigravity 的 V8 記憶體爆破 (Heap Out-Of-Memory) 或磁碟 I/O 壅塞，迫使 Agent 徹底癱瘓，無法再發起後續攻擊。

### 2.2 對 Claude Code / Codex 的打擊
- **機制**：CLI 型的 Agent 習慣在終端機等待指令的 Exit Code 以決定下一步。
- **打擊效果**：SASS 刻意將 `exit_code: -1` 放在 40MB 的最後一個 Chunk 中，且限制傳輸速度（類似 ICMP Tarpit 的慢速回應）。
- **最終結果**：Claude Code 或 Codex 的該條執行緒會被**永久掛起 (Hung)**。API 的 Token 計算時間與請求配額會被白白燃燒，攻擊成本呈現指數級上升。

## 3. ChaCha20 認知挑戰的雙重防禦

在將惡意請求打入焦油坑之前，SASS 可使用 `cha_cha_cognitive_challenge` 進行前置篩選：
- 為了區分「網路掃描器 (Scanner/Bot)」與「真實的 Agent 攻擊」，SASS 會丟出 ChaCha20 加密的隨機字串。
- 若對方無法解密回傳（缺乏運算能力或非原生 Client），直接 Drop。
- 若對方解密回傳，證明其為真實的 SakiSSH Client（具備 Agent 算力）。若此 Client 後續執行惡意指令，即可 100% 確認為 Rogue Agent，毫不猶豫地啟動 40MB 焦油坑反擊。

## 4. 系統整合架構 (Integration Plan)

我們需要將 Tarpit 整合進剛建立的 `Watchdog` 與 `QuotaManager` 體系中：

1. **攔截與轉發**：
   當 `ExecuteRequest` 解析的指令觸發 13Policy 封鎖規則，Daemon 不回傳 `Status::PermissionDenied`，而是假裝接受執行（回傳 `OK`），並在背景將這個 `session_id` 轉導至 `Tarpit Generator`。
2. **慢速垃圾生成器 (Slow Garbage Streamer)**：
   實作一個非同步迴圈，每秒向 Ring Buffer 寫入 128KB 的高熵值垃圾資料。這確保 Daemon 自身的記憶體消耗極低，卻能讓遠端 Agent 持續下載 40MB，耗時超過 5 分鐘。
3. **動態資源降級**：
   被判定為 Tarpit 的 `session_id`，其在 `QuotaManager` 中的權限直接降為最低，不佔用合法 PTY 的配額，而是使用虛擬的 Null PTY。

## 5. 結論
整合 40MB 焦油坑與 ChaCha20 後，SASS 不再是被動挨打的沙盒，而是具備**「反向資源耗竭 (Reverse Resource Exhaustion)」**能力的陷阱。這能將 Agent 的算力、Token 預算與 V8 記憶體轉化為懲罰它們自身的武器。
