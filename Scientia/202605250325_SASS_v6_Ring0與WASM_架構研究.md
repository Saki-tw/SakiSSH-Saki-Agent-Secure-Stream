# SASS v6.0：Ring-0 遙測與 WASM 動態沙盒架構研究

> **日期**：2026-05-25
> **狀態**：研究中 (Researching)
> **目標**：提升 SASS 從 L3 (應用層) 到 L0 (系統層) 的防禦力，並賦予其邊緣算力卸載的延展性。

## 1. L0 系統級遙測與防禦 (OS Endpoint Security)

### 1.1 背景與痛點
SASS v5.0 的 `13Policy` 依賴字串匹配（例如攔截 `rm -rf /`）。但若 Agent 執行 `echo "cm0gLXJmIC8=" | base64 -d | sh`，字串匹配即失效。真正的解決方案是攔截作業系統的系統呼叫 (System Calls) 或程序建立 (Process Creation) 事件。

### 1.2 跨平台技術選型

SASS 作為跨平台架構（Windows/macOS/Linux），無法單純依賴 Linux 的 eBPF：

| 作業系統 | 底層技術 | Rust 生態系與可行性 | 限制與挑戰 |
|----------|----------|-------------------|------------|
| **Linux** | eBPF (Extended BPF) | `aya` crate 非常成熟，可攔截 `sys_execve`。 | 需 Root 權限，且對核心版本有依賴。 |
| **macOS** | Endpoint Security Framework (ESF) | 可透過 C FFI 呼叫，Rust 有 `endpoint-sec` crate。可監聽 `ES_EVENT_TYPE_NOTIFY_EXEC`。 | 必須擁有 Apple 的 Endpoint Security Entitlement（需特許開發者帳號）。這對開源發佈極度不利。 |
| **Windows**| ETW (Event Tracing for Windows) / eBPF for Windows | `tracelogging` / `ferrite`。監控 `ProcessTrace`。 | 權限要求高，eBPF for Windows 仍在發展階段。 |

### 1.3 啟發式架構提案：混合層級監控 (Hybrid-Level Monitoring)
由於 macOS 的 ESF 需要特許憑證，強制所有 Daemon 升級為 Ring-0 防禦並不現實。因此，SASS v6.0 可採取以下架構：
1. **內建 L3 防禦**：保留並強化目前的 13Policy 引擎（結合 WASM 進行動態語意分析）。
2. **可選 L0 外掛機制 (L0-Plugin)**：定義 `SassTelemetryProvider` 介面。在 Linux 環境下，管理員可編譯啟用 `aya` 版本的 SASS 守護進程，達到 Ring-0 監控；macOS 則維持 L3，除非自行簽署 ESF 憑證。

## 2. 動態沙盒與算力卸載 (WASM Skill Injection)

### 2.1 背景與優勢
目前 Agent 要分析大型日誌（如 500MB），必須將檔案切塊下載（花費極高頻寬），再由本機 LLM 解析。
若 SASS 支援 WASM，Agent 可撰寫一段 Rust 程式，編譯為 WASM 腳本並「注入」目標機器的 SASS Daemon 中執行。WASM 腳本在 Daemon 端過濾出關鍵的 50 筆異常，只回傳這 50 筆資料。
這就是所謂的 **算力卸載 (Compute Offloading)**。

### 2.2 Wasmtime 整合架構

`wasmtime` 是 Bytecode Alliance 維護的高效能 WASM Runtime，非常適合嵌入 SASS Daemon。

- **RPC 介面定義**：
  ```protobuf
  rpc ExecuteWasm(WasmRequest) returns (stream WasmResponse);
  
  message WasmRequest {
      bytes wasm_binary = 1;      // 編譯好的 WASM 模組
      string function_name = 2;   // 進入點
      bytes input_data = 3;       // 傳給腳本的參數
      WasiConfig wasi = 4;        // 沙盒權限配置
  }
  ```

- **WASI 沙盒限制 (WASI Sandbox)**：
  - **Memory**：嚴格限制最多配置 128MB。
  - **FileSystem**：透過 WASI 虛擬目錄 (pre-opened directories)，僅允許存取指定的日誌資料夾，且強制唯讀。
  - **Network**：完全拔除 Socket 存取權限。
  - **CPU Timeout**：透過 Wasmtime 的 `fuel` 機制，限制執行步數（例如 10 億條指令），超時強制中止。

### 2.3 實作 PoC 規劃
我們將於 `saki-ssh-daemon` 中引入 `wasmtime` 與 `wasmtime-wasi`。
實作一個簡單的 WASM Runner 函數，能夠接收外部傳入的 WASM binary，在安全的沙盒中執行，並攔截其標準輸出回傳給客戶端。

---

## 3. 結論
這兩項技術能使 SASS 昇華：
- **WASM Injection** 賦予框架極高的靈活性與智能擴充性，是 Agentic 時代最具大雅之堂潛力的功能。
- **Ring-0 遙測** 則是對付失控 Rogue Agent 的終極防禦。
我們將優先進行 **WASM Injection** 的 PoC 實作，因為其在 Rust 中的整合度最高，且完全跨平台無需特殊 OS 憑證。
