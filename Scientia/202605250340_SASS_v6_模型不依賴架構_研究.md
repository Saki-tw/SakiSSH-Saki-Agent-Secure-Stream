# SASS v6.0：模型不依賴 (Model-Independent) 架構核心哲學研究

> **日期**：2026-05-25
> **狀態**：研究中 (Researching)
> **核心前提**：所有架構進化（Ring-0, WASM, ZKP, Hardware Trust, P2P）必須絕對與底層 LLM（Gemini, Claude, GPT 等）解耦。基礎設施不應預設任何「模型特性」，而應提供確定性 (Deterministic)、泛用性且極端安全的底層算力與網路接口。

## 1. 什麼是「模型不依賴」？

在 Agentic 開發領域，許多框架依賴特定模型的 Tool Use API 或 Context 特性（例如依賴長文本能力來塞入巨量系統提示詞）。
**SASS 的「模型不依賴」哲學定義**：
1. **無狀態 (Stateless)**：SASS 守護進程不在乎是誰下達指令——無論是 Gemini CLI、Claude Code，甚至是一般的 CI/CD Runner 或人類工程師，只要持有合法的 ED25519 憑證 / ZKP 證明，行為即一致。
2. **確定性防禦 (Deterministic Defense)**：不使用 LLM 或 AI Agent 來進行「意圖分析」或「威脅狩獵」。防禦機制必須基於數學密碼學（ChaCha20, ZKP）或 OS 核心機制（eBPF, 系統呼叫）。
3. **標準化算力介面**：不依賴 LLM 當場生成 brittle（脆弱）的 shell scripts，而是提供標準化的 WASM ABI 介面。

---

## 2. 五大方向的「模型不依賴」演繹

### 2.1 eBPF Ring-0 遙測：確定性執法 (Deterministic Enforcement)
- **非依賴性**：許多安全框架會將日誌送給 LLM 判斷是否有害（這高度依賴模型智商與幻覺機率）。SASS v6.0 的 eBPF 探針**不依賴 AI 判斷**。
- **作法**：透過 eBPF 攔截 `sys_execve` 與 `sys_openat`，配合寫死的、由管理員定義的 Rust 規則引擎（或 WASM 規則檔）。不符合 Capability ACL 的行為，由 OS 核心直接 Drop，達到 100% 確定性。

### 2.2 WASM 動態沙盒：標準化 ABI 與跨語言支援
- **非依賴性**：LLM 的輸出極易受提示詞與訓練數據影響，生成的 Python 腳本可能因目標機器的環境（如缺套件）而失敗。
- **作法**：WASM 是**不可變的二進位標準 (Immutable Binary Standard)**。不論 Agent 是 Gemini 還是 Claude，只要它能產出或調用預先編譯好的標準 WASM 模組（例如過濾敏感詞、計算 Hash、提取 JSON 陣列），目標機器上的 `wasmtime` 就能確保執行結果絕對一致。Agent 的任務從「撰寫可能失敗的腳本」轉變為「拼裝標準化的 WASM 算力積木」。

### 2.3 硬體級信任根 & ZKP 權限驗證：數學絕對性
- **非依賴性**：不依賴模型的「記憶力」來保管密碼，也不需要模型在 Context 中傳遞敏感的 API Key。
- **作法**：
  - **TPM/SEP**：密鑰直接綁死在實體機器上。模型被駭或被提詞注入 (Prompt Injection)，也無法提取私鑰。
  - **ZKP**：模型只需在本地提供 Proof。對模型而言，這就是一個「呼叫本地 ZK 產生器並將結果送到遠端」的簡單 Tool Call，大幅降低模型處理敏感資料的認知負擔。

### 2.4 P2P Mesh 網格：去中心化基礎設施
- **非依賴性**：不依賴中心化的 Orchestrator Agent 來轉發訊息。
- **作法**：SASS 節點透過 `libp2p` 建立 Kademlia DHT 網路。若 Model A (Gemini) 需要操作機器 X，但機器 X 躲在 NAT 後方，Model A 可以直接請 SASS 客戶端向 Mesh 網路廣播路由請求，透過節點 Y 進行中繼。網路拓樸的管理完全下放給底層 P2P 協定，模型只需要下達「連線到 Peer ID: XXX」的指令。

---

## 3. SASS v6.0 的終極定位

在「模型不依賴」的前提下，SASS v6.0 將不再只是 Agent 的附屬工具，而是進化為：
**「專為 AI Agent 設計的跨主機、零信任、分散式 Serverless 基礎設施 (Zero-Trust Serverless Infrastructure for AI Agents)」**

1. **對 Agent 而言**：SASS 是一個黑盒子，只要給它標準的 WASM payload 和目標 Peer ID，它就會安全、可靠地傳回結果。
2. **對資安團隊而言**：SASS 是一套具備硬體級信任與 eBPF L0 遙測的堅固堡壘，完全不需要擔心裡面的 Agent 發瘋或幻覺。

## 4. 下一步：核心介面抽象化
為貫徹此原則，下一階段的架構設計必須確保 gRPC 介面 (Protobuf) 完全無涉於 LLM 的上下文邏輯，所有的 API 設計（如 `ExecuteWasm` 的 WASI Config）皆應以嚴格的型別系統定義，杜絕模糊的自然語言參數。
