## 1. Aumann-Serrano 風險指標

Robert Aumann（2005 年諾貝爾經濟學獎）與 Roberto Serrano 在 2008 年發表的 "An Economic Index of Riskiness" 定義了一種風險排序方法：

> 一個賭局的風險度可以被「擠」出來——它介於排名第二與第三之間。不需要知道完整的概率密度函數，只需要相對排序。

## 2. 對應到 SASS 的 AES 宣稱

### 方法：展開非預期行為、比較所有預期行為

SASS 的 AES（Almost Everywhere Superior）宣稱不需要：
- ❌ 窮舉所有可能的攻擊（不可能且會發散）
- ❌ 計算概率密度函數的面積
- ❌ 證明安全性的絕對值

SASS 的 AES 宣稱只需要：
- ✅ 展開兩個版本的非預期行為分支
- ✅ 比較分支數量和期望值
- ✅ 展示「少了分支，期望值不變」

### 具體範例

```
SASS v1.3（無 Vi Swap）：
  Agent 機率空間展開 = 100 個分支
  期望值損失 = E[L] = 2MB 資料被非預期取得

SASS v1.4（有 Vi Swap）：
  Agent 機率空間展開 = 99 個分支
  期望值損失 = E[L] = 2MB（相同）

∵ 分支數 99 < 100，E[L] 相同
∴ F_{v1.4} SSD F_{v1.3}
∴ SASS v1.4 Almost Everywhere Superior 於 SASS v1.3  ∎
```

### 為什麼少了一個分支？

以 Vi Swap 為例：在 v1.3 中，一個已認證 Agent 嘗試越界指令時，Daemon 回傳錯誤訊息 → Agent 解析錯誤 → Agent 嘗試變形指令 → 這個「嘗試-失敗-重試」循環是一個**額外的分支**，帶有非零的損失風險。

在 v1.4 中，同一情境觸發 Vi Swap → Agent 陷入假 vi 畫面 → Agent 停止生成 → **該分支被消滅**。期望值不變（Agent 最終都無法執行越界指令），但分支少了一個。

## 3. 方法的收斂性

雖然無法窮舉所有可能的 Agent 行為，但這個比較方法**不會發散**：

1. 每個版本迭代都可以指出「消滅了哪些分支」
2. 每個被消滅的分支都可以驗證：「期望值是否改變？」
3. 如果期望值不變且分支減少 → SSD 成立
4. 如果期望值也降低 → 更強的優越性（FSD, First-order Stochastic Dominance）

這是一個**構造性證明**，不是存在性證明——我們不需要聲稱「對所有情況都更好」，我們只需要展示「對這個具體情境少了一個分支」。

## 4. 逆向方法：從風險參考擠出殘餘

Aumann-Serrano 的方法還有一個對偶性：

> 對應到有期望值的任何東西，還能用這個方法逆向擠出「風險的參考」→「風險以外的殘餘」。

在 SASS 中：
- **風險參考** = 已知的分支（展開的非預期行為）
- **殘餘** = 未知的分支（未被展開的非預期行為）
- 7 層 Safety Gradient 的每一層都在**壓縮殘餘的上界**

## 5. IETF 審查的務實考量

IETF 不太可能計較：
- 概率密度函數的具體面積
- 風險指標的數值精度

IETF 會看的是：
- 宣稱是否自洽
- 是否有明確的比較基準（v1.3 vs v1.4）
- 是否有構造性的論證

AES 的宣稱完全滿足這三點。如果需要數學支撐，可以直接引用 Aumann & Serrano (2008) 的框架，略改其 LaTeX 符號即可。人家已經拿了諾貝爾獎，方法論的正確性不需要我們再證明。

## 6. 關鍵澄清：「小」不是值

> SASS 損失的「小」並不是值，而是一個比較。

這是整個 AES 宣稱最重要的一句話。我們從不說「SASS 的損失 = 0.003」——這沒有意義。我們只說「SASS v1.4 比 v1.4 的上一個版本少了一個分支，期望值不變」。這就是二階隨機優越性的全部內容。
